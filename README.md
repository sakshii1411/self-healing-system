# Self-Healing Distributed System with ML Anomaly Detection

A production-grade self-monitoring and self-healing distributed system built with Python, FastAPI, Docker, Prometheus, Grafana, and Isolation Forest.

## Features

- **Two microservices** (`user-service` + `order-service`) with real inter-service communication
- **Structured JSON logging** + **Prometheus metrics** (Golden Signals)
- **Real-time monitoring** with Prometheus + Grafana dashboard
- **ML-powered anomaly detection** using Isolation Forest trained on live system data
- **Decision Engine** that detects anomalies (error rate, latency, CPU spikes)
- **Response Engine** that automatically takes corrective actions
- Fully containerized with Docker Compose

## Architecture
Microservices (user + order)
↓
Logs + Metrics
↓
Prometheus
↓
Grafana Dashboard
↓
ML Anomaly Detector (Isolation Forest)
↓
Decision Engine
↓
Response Engine (Automation)
text## Tech Stack

- **Backend**: Python + FastAPI
- **Observability**: Prometheus, Grafana
- **ML**: scikit-learn (Isolation Forest)
- **Containerization**: Docker + Docker Compose
- **Automation**: Rule-based + ML-driven actions

## How to Run

```bash
# Clone the repo
git clone <your-repo-url>
cd self-healing-system

# Start the entire system
docker compose up --build -d

# Access services:
# - User Service: http://localhost:8001
# - Order Service: http://localhost:8002
# - Prometheus: http://localhost:9091
# - Grafana: http://localhost:3001 (admin/admin)
# - Decision Engine: http://localhost:8003/decide
# - Response Engine: http://localhost:8004/act
# - ML Detector: http://localhost:8005/train and /detect
Key Endpoints

GET /orders → Triggers inter-service call + failures
GET /metrics → Prometheus metrics from each service
GET /decide → Decision Engine (rule-based)
GET /detect → ML Anomaly Detection
GET /act → Response Engine (automation)
